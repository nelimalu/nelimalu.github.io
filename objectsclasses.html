[comp sci] / objects & classes

objects and classes are the basis of Java programming
classes and objects are difficult to understand, so don’t worry if it takes a while!

/What is a class?

A class is a template for an object
it will specify what attributes something you’re trying to code will have
for example, a dog will have a set of attributes including

  • name
  • breed
  • age

To give these attributes specific values, we need to create an object

/What is an object?

An object is a specific instance of a class
a common analogy is a cookie cutter
a cookie cutter has a shape, and each cookie made from the cookie cutter will have that same shape, but they are each individual cookies (or objects)
When we create an object, we can give each attribute a specific value
for example, using the attributes from above

  • name = Randall
  • breed = Husky
  • age = 6

we can create as many objects (individual dogs with specific attributes) as we want from a class (the template of the dog, outlining that it will have a name, breed, and age)

How to write a class

here is an example of a class using the dog example above 

public class Dog {
	String name;
	String breed;
	int age;
}


To create an object of the Dog class, and give the dog a specific name, breed and age, we need a constructor.
A constructor is a unique method that called when a new object is being created.
It is always public, has no return type, and has the same name as the class (case sensitive)
Here is an example of a constructor for the dog class 

public class Dog {
	String name;
	String breed;
	int age;

	public Dog(String n, String b, int a) {
		name = n;
		breed = b;
		age = a;
	}
}

For each attribute of the class, a parameter is added to the constructor
make sure that the name of the parameter is different from the name of the attribute
in the future we can use the this keyword to avoid this naming issue

How to create an object

for most programs, we will have a separate driver class with a main method
a driver class usually only has one method, which is the main method
this is where the code will begin executing
Here is an example of a driver class 

public class Driver {
	public static void main(String[] args) {
		System.out.println("hello!");
	}
}


The driver class must be saved in the same folder as your other class, in this example, the Dog class
Here is an example of your file structure 

DogProject
├── Driver.java
└── Dog.java


We will create a dog object within the driver class
Here is an example of creating an object 

public class Driver {
	public static void main(String[] args) {

		Dog my_dog_object = new Dog("Randall", "Husky", 6);

	}
}

notice that `Dog` is being used as a data type of this variable
this is a very important concept to understand: ***classes are essentially just data types***
the variable `my_dog_object` is a variable of type `Dog`
To create a new object (an **instance** of the dog class), we use the `new` keyword
We then write the name of the class, and in brackets, write all the parameters of the constructor
notice that in the constructor written above, we specified the following: 

public Dog(String n, String b, int a) {  // pay attention to the order of
	name = n;                            // parameters here
	breed = b;
	age = a;
}

the order in which we wrote the parameters in the constructor is the order in which we enter the values of the attributes when we create a new object
In this example, the order is `name` , `breed` , `age`
This is why when we create the object, we do `new Dog("Randall", "Husky", 6)`
we can also create multiple objects from the same class

Manipulating objects

we already saw how objects have attributes
objects can also have methods
methods are blocks of code that manipulate the attributes of an object
To understand how methods work, let’s consider another example: a bank account 

public class BankAccount {
	String accountName;
	double balance;
	double interestRate;
}

a bank account may have some functions, like depositing money, withdrawing money, or applying interest
we can create a method for each of these 

public class BankAccount {
	String accountName;
	double balance;
	double interestRate;

	// constructor
	public BankAccount(String a, double b, double i) {
		accountName = a;
		balance = b;
		interestRate = i;
	}

	public void withdraw(double amount) {
		balance -= amount;  // subtract the amount from the balance
	}

	public void deposit(double amount) {
		balance += amount;  // add the amount to the balance
	}

	public void applyInterest() {
		balance *= (1 + interestRate);  // add one so it increases by the interest amount
	}
}

now let’s create some objects (or instances) of the BankAccount
We can print the attributes of specific objects by using `.` 

public class Driver {
	public static void main(String[] args) {

		// bank account with name luka, $100 in the bank, and 5% interest rate
		BankAccount LukaAccount = new BankAccount("Luka", 100, 0.05);
		// bank account with name Krasteva, $153.67 in the bank, and 2% interest rate
		BankAccount KrastevaAccount = new BankAccount("Krasteva", 153.67, 0.02);

		System.out.println(LukaAccount.accountName);
		System.out.println(KrastevaAccount.balance);
	}
}

/*
OUTPUT

Luka
153.67
*/
to run methods of an object, we simply use `.methodName()` and add any parameters in the brackets 
public class Driver {
	public static void main(String[] args) {
		BankAccount LukaAccount = new BankAccount("Luka", 100.0, 0.05);
		BankAccount KrastevaAccount = new BankAccount("Krasteva", 153.67, 0.02);

		System.out.println(LukaAccount.balance);
		LukaAccount.deposit(25.0);
		System.out.println(LukaAccount.balance);
		System.out.println(KrastevaAccount.balance);
	}
}

/*
OUTPUT

100.0
125.0
153.67
*/

this may be kind of obvious, but notice how running .deposit on Luka’s account does not affect the balance of Krasteva’s account, even though they are objects of the same class
you can probably imagine how the other methods work
