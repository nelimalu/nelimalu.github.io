[comp sci] / javadoc comments

Javadoc comments are html snippets that are used to summarize a class, method, or attribute

what a javadoc comment includes:

  • brief summary
  • parameters
  • return value

/Why do we use javadocs?

Documentation (where the doc comes from in javadoc) is a very important part of code, and is used to help others use and understand code that you write.
Usually, when someone makes a program, it will feature many classes with many methods. It’s not feasible for another programmer to go through your code and find the function of each method by looking at comments.
Documentation is a place where all methods and classes are summarized so programmers can easily find and use specific methods.
Since documentation is a generally boring task, where you’ve finished writing the code and now you’re just writing descriptions and explanations of methods, the javadoc was made!
Javadocs make it so that a programmer writing in Java can write their documentation as they’re writing their code, so they don’t have to do it later
These javadoc comments can then be parsed by a fancy javadoc generator, and create your own documentation website for you
The website aspect is the reason we can use html within our javadocs
Essentially, for a small program with one or two files, javadocs are useless, but they are great practice for if you may write a large scale library in the future.

/Example of Documentation

https://docs.gl/gl2/glAccum

OpenGL is a famous program used to create 3D graphics
Usually, large programs such as this are actually a compilation of thousands of small programs which together are called a library
In this example, we have a random method called glAccum() which is not a very descriptive name, and would be hard for a programmer to figure out what it’s used for
Luckily, the documentation explains every aspect of this method
It lists the parameters, return values, and how they’re used
It also has a description to help you understand how this method is used, and when to use it

Writing a Javadoc Comment

A Javadoc comment is a multiline comment
The first line has two asterisks *
All following lines have an asterisk preceding the line


Example: 
/**
 * this is an example javadoc comment 
 */

Javadoc comments can also include html tags, because javadocs get compiled into HTML.
This can be used to emphasize parts of a javadoc comment 

/**
 * <h1>This text is going to appear bigger</h1>
 * than this text
 */

Javadoc for a class

Class javadocs are a more professional version of "four top line comments" taught in previous ICS classes.
This comment summarizes the function of a class
It also includes the author (your name), the date, as well as a version number.
Put an @ symbol in front of your name and version number

/More on version numbers

Version numbers can be confusing, but if you’re writing a big program that you may be working on for months, every time you make a big change to your code you would change the version number.
Think how minecraft (a java program) has version numbers like 1.0, 1.8.9, 1.19, etc.
It’s also common to have a version like beta 1.0
Prefixes like alpha or beta are used to show a user that your program is not complete, and may have frequent bugs.

Example: 
/**
 * <h1>This text is going to appear bigger</h1>
 * than this text
 */

Javadoc for a method

A method javadoc summarizes what a method is supposed to do.
It specifies the parameters and the return value, with a short description for each.
Remember to put an @ symbol in front of parameter and return

Example: 
/**
 * add method to easily add two numbers
 * 
 * @param a First integer to be added
 * @param b Second integer to be added
 * @return The sum of a and b
 */
public int add(int a, int b) {
	return a + b;
}

Javadoc for an attribute (field)

This is usually one line, and simply explains the use of the attribute.

Example: 
/** The age of the person */
private int age;
